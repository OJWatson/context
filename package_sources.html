Information about non-CRAN packages

Usage

package_sources(cran = NULL, repos = NULL, github = NULL, bitbucket = NULL, local = NULL, expire = 1)

Arguments

cran
Use this to set the CRAN mirror. The default will use the Rstudio CRAN mirror which is usually nice and fast.
repos
Character vector of additional reposotories to use. Specify whatever URL would normally work with install.packages. In addition, specify URL of the form drat:// to install from drat repositories.
github, bitbucket
Character vector of github or bitbucket packages to install. The format is similar to devtools::install_github, being username/repo[/subdir][@ref] (note that pull requests are not supported in contrast with devtools).
local
Character vector of paths to packages on your local computer to make available. These can be directories or built tar.gz files.
expire
Number of days after which to expire packages in the archive. After this, packages will be fetched again. Fractional days or difftime objects are fine.

Description

Specify information about locations to find non-CRAN packages. This will typically be saved with a context. As part of the context generation process, any packages not in a repository (CRAN or other CRAN-like repository such as a drat reposotory) will be copied into a local drat repository that will be available to a process that is restoring the context.
