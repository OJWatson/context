Install packages

Usage

install_packages(packages, sources = package_sources(), ..., error = TRUE, lock_wait = TRUE)

Arguments

packages
A character vector of packages to install
sources
Source information from package_sources.
...
Additional arguments passed through to install.packages.
error
Throw an error if package installation fails.
lock_wait
Lock the directory while installing, or wait for a second installation to complete. In contrast with install.packages, the lock is always library-wide. Catastrophoic failure (e.g., out of memory error during package installation) could leave a lockfile present. In that case, delete the file .lock_context from the first entry in .libPaths().

Description

Install packages, using source information from package_sources. In contrast with install.packages this function can install packages in a new local repository (used to isolate packages from the rest of the system) and can install packages originally found on github and bitbucket by creating a transient drat repository. In addition, install_packages_missing only installs missing packages only. Also in contrast with install.packages, these function will throw an error if package installation fails (unless error=FALSE is set).

Warning

Because of the possibility of confusion with install.packages, this function may be renamed shortly.
