Save and reload contexts

Usage

context_save(packages = NULL, sources = NULL, auto = FALSE, package_sources = NULL, envir = parent.frame(), root = tempdir())


context_load(handle, install = TRUE, envir = .GlobalEnv, ...)


context_read(handle)

Arguments

packages
A character vector of packages (or NULL) if no packages are to be loaded.
sources
A character vector of source files to load.
auto
Attempt to create the context automatically. In this case, do not specify either packages or sources.
package_sources
Optional information about where to find non-CRAN packages. See package_sources.
envir
The current environment. This is used to copy local enviroments around, as these are needed even in the case of non-automatic contexts. For context_load this is the environment into which the global environment is copied. Specify a non-global environment here to avoid clobbering the workspace, but at the risk that some environments may not restore exactly as desired.
root
Root directory to store and retrieve files. Files will be added to the contexts subdirectory of this path. This will change later to support alternative ways of saving files. The default puts files into the temp directory of this R instance; this directory will be deleted on session exit so it not an appropriate place to store files for later use.
handle
A context_handle object returned by context_save.
install
Install missing packages?
...
Additional arguments passed through to install_packages if it is used.

Description

Save and reload contexts. Contexts consist of packages and sources, or (if auto) is TRUE) a copy of the current global environment. Environments saved with context_save can be reloaded with context_load; note that doing this will probably alter the search path by loading any number of packages.

Details

The context_read function simply reads the context, but does not load it. It is mostly useful for debugging.
